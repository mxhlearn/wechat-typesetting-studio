# Originality And Voice

Use this file when the user wants the article to feel less like AI copy and more like a real finished piece.

Also use it whenever reference articles, benchmark samples, or source drafts are involved.

## Core Rule

Do not aim for "smooth". Aim for "specific, natural, and worth reading".

If a paragraph sounds fluent but could belong to almost any article on the topic, it is too generic.

Every paragraph must add one of these:

- fact
- judgment
- step
- example
- consequence
- contrast

If it adds none, cut it.

Use a stricter rule for public-account drafts: if a sentence only increases politeness, atmosphere, or completeness, cut it unless it changes reader action or judgment.

## Anti-AI Tone Rules

Avoid these pattern types unless the user explicitly wants them:

- generic era-opening line
- generic trend-opening line
- "many people ask this" opening
- "this article will explain several aspects" opening
- "let us take a look together" transition
- generic empathy-close line
- empty summary transition
- stacked benefit claims before showing the actual method
- repeated "this matters because" paragraphs
- balanced but empty "on the one hand / on the other hand" phrasing
- conclusions that only restate the title
- invented certainty around facts the source did not provide
- decorative adjectives that do not change the reader's understanding

Prefer:

- a direct situation
- a concrete problem
- a clear task
- a clean transition that actually moves the article forward
- a verifiable detail
- a specific editorial judgment
- a narrower claim that can be defended

## Natural Chinese Rhythm

Good public-account Chinese often feels:

- slightly spoken but not loose
- specific but not stiff
- concise without sounding chopped up
- willing to cut explanatory padding

Do not over-polish every sentence until the piece loses texture.

Allow:

- short standalone sentences when they sharpen a point
- occasional bluntness when the topic is operational
- simple phrasing over elegant emptiness

Prefer exact verbs:

- `配置` over `进行配置`
- `打开` over `进入到相关页面`
- `验证` over `检查是否成功完成`
- `删掉` over `进行删除处理`
- `保留` over `予以保留`

## Originality Rules

When using a source draft:

- preserve the user's real point
- rewrite from meaning, not from sentence order
- do not keep weak original phrasing just because it already exists

When using reference articles:

- extract structure
- extract promise strategy
- extract pacing
- extract packaging moves

Do not reuse:

- distinctive metaphors
- recognizable transitions
- sentence-level cadence copied too closely
- section order that mirrors a reference for no good reason

If the draft depends on references:

1. read the references
2. summarize the reusable method in your own words
3. stop looking at the reference phrasing
4. draft from the method summary

## Signs A Paragraph Still Sounds AI-Written

Rewrite when you see:

- every sentence sounds equally polished
- the paragraph contains no real choice or viewpoint
- transitions are smooth but content is thin
- the paragraph explains obvious things the reader already knows
- the wording sounds safe in a way no real writer would naturally choose

## Signs A Paragraph Feels Too Close To A Reference

Rewrite when you see:

- same section purpose in the same order
- same rhetorical setup and payoff
- same keyword cluster in near-identical spacing
- same emotional rise and conclusion

If this happens, rebuild the paragraph from scratch using only:

- the factual point
- the intended reader outcome
- the chosen article structure

## Practical Rewrite Moves

Use these moves to de-template the draft:

- replace abstract summary with one concrete sentence
- merge two polite transitions into one direct line
- cut the sentence that explains what the next paragraph will say
- turn broad encouragement into operational clarity
- replace "this is important" with the actual reason it matters
- replace "many people" with the actual reader group
- replace "improve efficiency" with the specific saved step or reduced risk
- replace "worth noting" with the note itself
- replace "in summary" with the final action or judgment
- replace "obviously / inevitably / completely" with the actual condition or evidence

## Precision Pass

Run this pass before final delivery:

1. Circle every broad noun: 能力, 体验, 效率, 价值, 生态, 方案.
2. Replace it with the actual action, risk, result, or decision.
3. Cut every sentence that only announces structure.
4. Replace inflated claims with conditional claims.
5. Keep one strong judgment per section when the article is commentary or review.
6. Keep one verification point per critical step when the article is a tutorial.

Common replacements:

| Weak | Better |
| --- | --- |
| 提升使用体验 | 少切一次窗口 / 少改一次配置 / 更容易验证结果 |
| 降低门槛 | 第一次配置也能按步骤跑通 |
| 值得关注 | 会影响安装路径 / 写作节奏 / 发布前检查 |
| 赋能创作 | 帮你把草稿改成可发布文章 |
| 完整解决方案 | 这篇只解决安装、配置、验证三件事 |

## Sentence-Level Cut List

Cut or rewrite sentences that begin like:

- "在这个快速变化的时代"
- "随着技术的发展"
- "对于很多人来说"
- "我们不妨先来看看"
- "本文将带你了解"
- "值得一提的是"
- "总的来说"
- "这也说明了一个问题"

Keep them only when the rest of the sentence contains a concrete, non-generic claim.

## Final Voice Check

Before delivery, ask:

- does this sound like a person with a point, or a model trying to sound complete
- would a human editor cut this sentence as empty
- is any paragraph memorable for the right reason, or only well-formed
- if I remove the title, does the opening still prove the article knows what it is doing
- would this sentence survive a strict human editor
